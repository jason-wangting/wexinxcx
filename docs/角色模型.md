# 角色模型设计

## 1. 文档目标

定义系统中的最小角色边界、超级管理员判定规则与鉴权最小流程。

---

## 2. 当前范围

本设计当前关注：

- 最小角色定义
- 超级管理员来源与判定规则
- 鉴权最小流程

本阶段不展开完整权限点矩阵、菜单级权限映射和接口级 ACL 细则。

---

## 3. 核心结论

### 3.1 角色最小化

当前系统只保留两层身份：

- 超级管理员（`SUPER_ADMIN`）
- 普通用户（`USER`）

其中普通用户无需写入独立角色表；只要存在合法用户档案即可视为 `USER`。

### 3.2 数据隔离原则

数据隔离基于资源归属字段（如 `ownerId`、`storeId`、`userId`），不依赖 role。

---

## 4. 超级管理员规则

### 4.1 来源

- 超级管理员不允许人工分配
- 由代码中写死的特定 `userId` 判定

### 4.2 判定优先级

鉴权时优先判断是否命中超级管理员：

1. 命中固定 `userId` -> 视为 `SUPER_ADMIN`
2. 未命中 -> 视为 `USER`

---

## 5. 鉴权与授权最小流程

### 5.1 鉴权

1. 获取当前调用用户 `userId`
2. 判断是否命中超级管理员固定 `userId`
3. 若命中，按超级管理员权限放行
4. 若未命中，按普通用户权限放行

### 5.2 授权

当前阶段不维护 `user_roles`、`roles` 的业务授权流程。

说明：

- 业务能力差异与模式切换规则统一在 `docs/用户模型.md` 中定义
- 高风险操作的人工复核规范见 `docs/人工审核流程.md`

## 7. 已确认设计决策

1. 角色体系最小化为 `SUPER_ADMIN` + `USER`
2. 超级管理员由固定 `userId` 判定，且不允许人工分配
3. 普通用户以合法用户档案存在为准
4. 数据隔离基于资源归属字段，不依赖角色
