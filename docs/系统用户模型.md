# 系统用户模型

## 1. 文档目标

定义用户主体、微信身份绑定、默认角色与版本分流字段，支撑“管理版 / 码头版 / 普通用户版”的进入逻辑。

---

## 2. 核心对象

### 2.1 用户主体

- 用户是系统身份主实体
- 每个用户必须有唯一 `userId`
- `userId` 创建后不可修改

### 2.2 微信身份绑定

- 登录主键：`appId + openId`
- `unionId` 作为跨应用增强识别字段，可为空后补
- 手机号可选，不作为主身份键

---

## 3. 用户状态与版本字段

### 3.1 系统角色字段

全局角色按 [系统角色模型](./系统角色模型.md) 判定：

- `SUPER_ADMIN`
- `PROMOTER`
- `USER`

### 3.2 应用版本字段（建议）

建议在用户档案中维护：

- `available_app_versions`：可进入版本列表
- `default_app_version`：默认进入版本
- `current_app_version`：当前会话版本

版本枚举：

- `ADMIN_APP`（管理版）
- `TERMINAL_APP`（码头版）
- `NORMAL_APP`（普通用户版）

约束建议：

- `current_app_version` 必须属于 `available_app_versions`
- `default_app_version` 必须属于 `available_app_versions`

---

## 4. 首次进入分流规则

### 4.1 分流问题

首次进入时询问：

- 是否为码头店铺工作人员

### 4.2 分流结果

- 是：默认进入 `TERMINAL_APP`
- 否：默认进入 `NORMAL_APP`

### 4.3 管理版准入

- 命中 `SUPER_ADMIN` 规则后，才允许进入 `ADMIN_APP`
- 非超级管理员即使手动构造入口也必须拒绝

### 4.4 推广员入口

- 命中 `PROMOTER` 时，在“我的”页面展示推广员入口
- 该入口是功能入口，不改变用户主版本分流

---

## 5. 数据结构建议（云数据库）

### 5.1 `users`

```json
{
  "_id": "oid_xxxxx",
  "userId": "user_xxx_12345",
  "unionId": "oXxxx",
  "phone": "13800138000",
  "name": "张三",
  "avatar": "cloud://env-id.xxx/avatars/user_xxx_12345.jpg",
  "status": "ACTIVE",
  "available_app_versions": ["NORMAL_APP"],
  "default_app_version": "NORMAL_APP",
  "current_app_version": "NORMAL_APP",
  "createdAt": 1672531200000,
  "updatedAt": 1672531200000
}
```

索引建议：

- 唯一索引 `userId`
- 稀疏唯一索引 `unionId`
- 普通索引 `status`
- 普通索引 `current_app_version`

### 5.2 `wx_identities`

```json
{
  "_id": "oid_xxxxx",
  "userId": "user_xxx_12345",
  "appId": "wx123abc456def",
  "openId": "oXxxx_openid_abcdef",
  "platform": "MINIPROGRAM",
  "createdAt": 1672531200000,
  "updatedAt": 1672531200000
}
```

索引建议：

- 唯一索引 `(appId, openId)`
- 普通索引 `userId`

---

## 6. 云函数登录与分流流程

1. 通过微信登录拿到 `appId/openId/unionId`
2. 按 `(appId, openId)` 查询 `wx_identities`
3. 未命中则创建用户与绑定记录
4. 新用户执行首次分流问答，写入 `default_app_version`
5. 命中超级管理员规则时补充 `ADMIN_APP` 可用版本
6. 命中推广员授权时返回推广员入口标记
7. 最终返回 `userId + current_app_version + entry_flags`

---

## 7. 当前已确认规则

1. 新用户默认普通用户
2. 首次分流依据“是否码头店铺工作人员”
3. 管理版仅超级管理员可访问
4. 推广员由超级管理员分配后才展示入口
5. 角色判定、版本准入与店铺权限校验必须在后端完成
