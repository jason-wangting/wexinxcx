# 推广员设计（原销售经理文档）

## 1. 文档目标

定义推广员角色的来源、入口与能力边界。

---

## 2. 角色定位

- 推广员是系统全局角色之一（`PROMOTER`）
- 推广员由超级管理员分配
- 推广员不是独立主版本，而是附加功能入口

---

## 3. 入口规则

- 用户默认无推广员入口
- 当用户拥有有效 `PROMOTER` 角色时
- 在“我的”页面显示推广员入口

---

## 4. 授权规则

1. 仅超级管理员可授予或撤销推广员
2. 授权需幂等（重复授权不应产生多条有效记录）
3. 撤销后入口应即时隐藏
4. 授权变更需记录审计日志

---

## 5. 数据模型建议

可复用 `user_roles`：

- `user_id`
- `role_code`（固定 `PROMOTER`）
- `status`（`ACTIVE` / `INACTIVE`）
- `source`
- `created_by`
- `created_at`
- `updated_at`
