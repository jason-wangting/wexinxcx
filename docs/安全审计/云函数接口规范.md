# 安全审计云函数接口规范

## 1. 文档目标

定义安全审计体系相关云函数接口，统一请求协议、鉴权约束、幂等规则、错误码与返回结构，作为研发联调与测试验收基线。

适用范围：微信小程序 + 微信云开发（云函数 + 云数据库）。

---

## 2. 通用协议

## 2.1 请求信封

所有写接口必须使用统一请求信封：

```js
{
  request_id: string,           // 客户端生成，服务端兜底
  idempotency_key: string,      // 幂等键，写接口必填
  action_name: string,          // 动作名，如 CUSTOMER_UPDATE
  reason: string | null,        // 高风险动作必填
  payload: object               // 业务参数
}
```

## 2.2 响应信封

```js
{
  request_id: string,
  event_id: string | null,
  code: string,                 // OK 或错误码
  message: string,
  result: object | null,
  audit: {
    occurred_at: number
  } | null
}
```

## 2.3 通用错误码

- `OK`
- `INVALID_PARAM`
- `UNAUTHORIZED`
- `FORBIDDEN`
- `SCOPE_VIOLATION`
- `STATE_CONFLICT`
- `IDEMPOTENCY_CONFLICT`
- `AUDIT_WRITE_FAILED`
- `INTERNAL_ERROR`

## 3. 统一基础函数

## 3.1 secure_edit_gateway

用途：全部业务写操作唯一入口（建议以公共模块方式复用）。

入参：

```js
{
  request_id: string,
  idempotency_key: string,
  action_name: string,
  reason: string | null,
  target: {
    collection: string,
    primary_key: string,
    store_id: string | null,
    port_id: string | null,
    voyage_id: string | null
  },
  payload: object
}
```

处理步骤：

1. 构建鉴权上下文（`auth_context`）
2. 权限与作用域校验
3. 读取旧值并计算差异
4. 事务写业务数据
5. 事务写 `audit_events`
6. 更新幂等状态并返回

---

## 4. 审计域云函数清单

## 4.1 auditWriteProxy

用途：标准化代理写入，业务函数通过该函数或公共模块触发统一审计。

权限：内部服务调用，不直接暴露前端。

`payload` 关键字段：

- `target`
- `mutation`（业务写入描述）
- `risk_level`

返回：

- `event_id`
- `changed_fields`

## 4.2 auditQueryEvents

用途：按维度检索审计事件。

权限：
- `SUPER_ADMIN`：全量查询，无 `store_id` 限制
- `STORE_ADMIN`：服务端强制将 `target.store_id` 限定为调用者所属店铺
- `STORE_MEMBER`：返回 `FORBIDDEN`，不允许调用

`payload`：

```js
{
  filters: {
    operator_user_id?: string,
    target_collection?: string,
    target_primary_key?: string,
    request_id?: string,
    action_type?: string,
    time_range?: { start_at: number, end_at: number }
  },
  page_no: number,
  page_size: number,
  order_by: "occurred_at_desc" | "occurred_at_asc"
}
```

返回：

- `list`
- `total`
- `page_no`
- `page_size`

## 4.3 auditGetTimeline

用途：返回单对象完整变更时间线。

权限：
- `SUPER_ADMIN`：全量
- `STORE_ADMIN`：仅限 `target.store_id` 属于本店铺的对象
- `STORE_MEMBER`：返回 `FORBIDDEN`

`payload`：

```js
{
  target_collection: string,
  target_primary_key: string,
  start_at?: number,
  end_at?: number
}
```

返回：

- `timeline_events`（按 `occurred_at asc`）

## 4.4 auditVerifyIntegrity

用途：每日锚点核验。

权限：`SUPER_ADMIN`。

`payload`：

```js
{
  anchor_date?: string
}
```

返回：

```js
{
  valid: boolean,
  detail: object
}
```

## 5. 业务域必须接入的写接口规范

以下业务函数不允许直接写库，必须调用 `secure_edit_gateway`：

1. 客户管理
- `customerCreate`
- `customerUpdate`
- `customerChangeStatus`
- `customerBindUser`
- `customerUnbindUser`

2. 角色与成员管理
- `roleGrantPromoter`
- `roleRevokePromoter`
- `storeMemberGrant`
- `storeMemberRevoke`

3. 订单与收款
- `personSaleOrderUpdate`
- `itemSaleOrderUpdate`
- `collectionRecordWriteoff`
- `collectionRecordReverse`

4. 日志核验
- `auditVerifyIntegrity`

---

## 6. 高风险动作接口约束

高风险动作定义见 [动作编码字典](./动作编码字典.md) 中 `risk_level=HIGH` 的动作，接口必须满足：

1. `reason` 必填，最小长度建议 10
2. 非 `SUPER_ADMIN` 不允许越权执行

---

## 7. 幂等规则

1. 写接口必须传 `idempotency_key`
2. 同一 `idempotency_key` + 同一操作者 + 同一动作视为同一请求
3. 已成功请求返回首个成功结果摘要，不重复执行写入
4. 已失败请求可允许重试（建议变更幂等键）

推荐幂等键格式：

- `action_name:store_id:target_id:client_nonce`

---

## 8. 鉴权与作用域校验规范

每个写接口都应显式返回失败原因：

- 全局角色不足 -> `FORBIDDEN`
- 非店铺成员 -> `SCOPE_VIOLATION`
- 资源跨链路 -> `SCOPE_VIOLATION`
- 状态机不允许 -> `STATE_CONFLICT`

同时将校验摘要写入审计事件的 `permission_snapshot`。

---

## 9. 函数开发建议

1. 按领域拆函数，按能力复用网关
2. 日志与审计分层：运行日志用于排障，审计日志用于合规追溯
3. 云函数超时控制：写接口推荐 3-8 秒内完成
4. 对外返回不包含敏感原文，仅返回必要摘要

---

## 10. 联调与验收清单

1. 每个写接口可返回 `event_id`
2. `request_id` 可贯穿前端日志、函数日志、审计事件
3. 幂等重复请求不产生第二条业务副作用
4. 审计写失败时业务写入必须失败
5. 日志查询接口严格执行角色与店铺作用域过滤
