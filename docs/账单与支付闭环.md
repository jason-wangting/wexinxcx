# 账单与支付闭环

## 1. 文档目标

定义码头版销售记账、普通用户付款、支付回调与状态更新的闭环。

---

## 2. 当前主链路

1. 码头版录入销售订单与应收
2. 账单归属到港口、店铺、船只、航次与用户
3. 将账单推送给普通用户侧查看
4. 普通用户可全额或部分付款
5. 以支付回调结果更新账单状态
6. 线下场景支持人工核销

---

## 3. 当前已确认规则

1. 支持全额付款
2. 支持部分付款
3. 支持分次付款
4. 支持人工核销
5. 支付结果以后端回调为准
6. 平台不持有资金，仅维护业务状态

---

## 4. 设计原则

- 订单状态必须可重算（未付/部分付/已结清）
- 支付状态更新必须幂等
- 支付流水与订单主状态分离建模
- 全链路保留审计信息

---

## 5. 后续待细化

- 账单主表与流水表关系
- 退款、冲正、作废状态机
- 线下核销撤销策略
- 离线补传与回调并发的一致性策略
