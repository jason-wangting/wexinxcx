# 会员与优惠统一设计

## 1. 文档目标

设计一套可落地的店铺会员系统，满足以下核心需求：

- 支持月付与年付
- 支持多天体验
- 支持折扣码
- 支持优惠券
- 管理员可设置全局体验天数
- 管理员可设置全局优惠券

本设计基于微信小程序 + 微信云开发 + 云函数 + 云数据库（NoSQL）。

---

## 2. 业务范围

本设计覆盖：

- 会员套餐定义与上下架
- 店铺会员权益生效与到期
- 试用逻辑（全局配置）
- 优惠券与折扣码（全局配置）
- 下单、支付、回调、续费、退款
- 开票申请、审核、开票结果回写
- 功能闸门校验（店铺角色 + 会员激活）

不覆盖：

- 税务系统对接细节与第三方开票服务商 SDK 封装细节
- 复杂营销组合（多券叠加、阶梯返利）

---

## 3. 角色与权限

### 3.1 系统角色

- 超级管理员：配置全局参数、套餐、优惠券、折扣码
- 普通店铺管理员：为所属店铺开通/续费会员
- 店铺成员：只读会员状态，无配置权限

### 3.2 双闸门原则

所有关键业务操作必须同时通过：

1. `checkStorePermission(storeId, permission)`
2. `checkStoreActivation(storeId, featureCode)`

---

## 4. 关键业务规则

1. 店铺未激活时，仅可访问基础信息，不可新增船只/航次/销售记录。
2. 套餐默认至少包含：月付、年付。
3. 试用天数由管理员全局配置，例如 3 天、7 天、15 天。
4. 单店铺默认仅允许领取一次试用权益。
5. 优惠券由管理员全局创建，可配置有效期、门槛、总量、每用户上限。
6. 折扣码由管理员全局创建，可配置有效期、门槛、配额、每用户上限。
7. 单笔订单默认仅允许一种优惠：优惠券或折扣码二选一。
8. 支付结果以后端回调为准，前端支付结果只作展示。
9. 续费采用到期顺延：
   - 未过期续费：在当前 `expires_at` 基础上顺延
   - 已过期续费：从支付成功时间起算
10. 退款不回滚历史业务数据，仅影响未来权益状态与可用能力。
11. 支持开票：仅已支付订单可申请开票，且同一订单可开票金额不超过实付金额。
12. 支持退款：仅已支付订单可发起退款，退款总额不可超过实付金额。
13. 开票与退款均需审计留痕，并支持幂等重试。

## 5. 核心流程

### 5.1 新店铺试用流程

1. 店铺创建完成
2. 云函数读取全局配置 `global_trial_days`
3. 若试用开关开启且店铺未试用过：
   - 写入 `store_membership_entitlements`
   - `activation_status=ACTIVE`
   - `source=TRIAL`
   - `started_at=now`
   - `expires_at=now + global_trial_days`
4. 写入事件日志

### 5.2 开通/续费流程（月付/年付）

1. 店铺管理员选择套餐（月/年）
2. 可选择优惠券或输入折扣码
3. 云函数计算应付金额并创建订单
4. 前端拉起微信支付
5. 微信回调云函数验签并幂等处理
6. 订单置为 `PAID`，更新店铺权益到期时间
7. 核销优惠券/折扣码
8. 写入权益事件日志

### 5.3 到期与降级流程

1. 定时任务扫描即将到期和已到期权益
2. 到期后状态切换为 `EXPIRED`
3. 关键能力接口在激活校验时拒绝执行

### 5.4 开票流程

1. 店铺管理员在已支付订单上提交开票申请（抬头、税号、邮箱、金额等）
2. 云函数校验订单状态、可开票余额与申请参数
3. 创建开票单并进入 `PENDING_REVIEW` 或直接 `APPROVED`
4. 调用开票服务（或人工处理）并回写发票号、开票时间、发票链接
5. 开票单状态更新为 `INVOICED`，写入事件日志

### 5.5 退款流程

1. 店铺管理员发起退款申请（订单、退款金额、原因）
2. 云函数校验可退款余额与订单状态
3. 创建退款单并进入 `PENDING`
4. 调用微信退款接口，异步回调确认结果
5. 退款成功后更新订单退款汇总、权益状态（按策略）并写事件日志
6. 退款失败则记录失败原因并支持重试

## 7. 计价与优惠规则

### 7.1 价格计算

- `base_amount_fen = plan.price_fen`
- `discount_amount_fen = coupon 或 discount_code 计算值`
- `amount_fen = max(0, base_amount_fen - discount_amount_fen)`

### 7.2 优惠校验顺序

1. 校验套餐是否有效
2. 校验优惠类型（二选一）
3. 校验有效期、门槛、总量、用户限用
4. 先写使用记录 `LOCKED`
5. 支付成功改为 `USED`
6. 订单关闭/失败改为 `RELEASED`

---

## 8. 状态机

### 8.1 订单状态机

- `CREATED -> PAYING -> PAID`
- `CREATED/PAYING -> CLOSED`
- `PAID -> REFUNDED`
- 任意 -> `FAILED`

### 8.2 店铺会员状态机

- `INACTIVE -> ACTIVE`（试用发放或支付成功）
- `ACTIVE -> EXPIRED`（到期）
- `EXPIRED -> ACTIVE`（续费成功）

### 8.3 开票状态机

- `PENDING_REVIEW -> APPROVED -> INVOICED`
- `PENDING_REVIEW -> REJECTED`
- `APPROVED -> FAILED`

### 8.4 退款状态机

- `PENDING -> PROCESSING -> SUCCESS`
- `PENDING/PROCESSING -> FAILED`
- `PENDING -> CANCELED`

## 9. 云函数接口建议

### 9.1 管理端（超级管理员）

- `updateGlobalMembershipSettings`：设置全局试用天数、开关
- `createOrUpdatePlan`：管理月付/年付套餐
- `createOrUpdateGlobalCoupon`：管理全局优惠券
- `createOrUpdateDiscountCode`：管理全局折扣码

### 9.2 店铺端（店铺管理员）

- `applyTrialForStore`：申请或触发试用（按全局规则）
- `createMembershipOrder`：创建会员订单
- `queryMembershipStatus`：查询店铺会员状态

### 9.3 支付回调

- `membershipPaymentCallback`：验签、幂等、更新订单与权益、核销优惠

### 9.4 开票接口

- `createInvoiceApplication`：创建开票申请
- `reviewInvoiceApplication`：审核开票申请（管理员）
- `issueInvoice`：执行开票并回写结果
- `queryInvoiceList`：查询开票记录

### 9.5 退款接口

- `createRefundRequest`：创建退款申请
- `reviewRefundRequest`：审核退款申请（管理员）
- `executeRefund`：发起微信退款
- `membershipRefundCallback`：退款回调处理
- `queryRefundList`：查询退款记录

## 10. 幂等与一致性

1. 订单创建幂等：`idempotency_key = store_id + plan_id + user_id + date`
2. 回调幂等：以 `wx_out_trade_no + PAID` 判重
3. 优惠核销幂等：使用记录状态机必须单向流转
4. 权益更新与订单更新使用同一事务提交
5. 定时补偿：
   - 关闭超时未支付订单
   - 扫描并回收长期 `LOCKED` 的优惠使用记录
   - 扫描并推进到期权益状态
6. 开票幂等：同一 `idempotency_key` 只允许生成一条开票申请。
7. 退款幂等：同一 `wx_out_refund_no` 只允许一次成功退款。
8. 金额一致性：订单层维护 `refunded_amount_fen` 与 `invoiced_amount_fen` 汇总字段并在事务内更新。

## 12. 实施优先级

1. 先落 `membership_global_settings` + `membership_plans` + `membership_entitlements`
2. 再落订单与支付回调闭环（`membership_orders`）
3. 最后接入全局优惠券与折扣码
4. 管理端补齐“全局试用天数 + 全局优惠券”配置页面
5. 增加开票与退款管理页（申请、审核、结果查询）
