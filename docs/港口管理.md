# 港口管理

## 1. 文档目标

定义港口作为码头版顶层业务边界的管理规则，以及港口与店铺归属关系。

---

## 2. 领域定位

- 港口是码头版最上层实体
- 店铺必须归属于某个港口
- 一个港口下可有多个店铺
- 港口不直接承接销售交易

码头版链路：

`port -> store -> ship -> voyage -> person_sale_order -> item_sale_order`

---

## 3. 权限边界

- 港口由超级管理员统一管理
- 普通店铺用户不能创建或变更港口
- 港口变更属于高风险操作，必须写入审计日志，并通过新建或迁移流程处理

---

## 4. 核心规则

1. 店铺创建时必须选择港口
2. 港口一旦选中不可修改
3. 如需调整港口归属，应通过新建店铺或迁移流程实现
4. 港口停用后不得新建归属店铺

---

## 5. 数据结构建议

### 5.1 `ports`

- `_id`
- `port_id`
- `name`
- `region`
- `status`（`ACTIVE` / `INACTIVE`）
- `created_by` （用户ID，值为 `user.user_id`）
- `created_at`
- `updated_at`

索引建议：

- `status`
- `name`

### 5.2 与 `stores` 的关系

- `stores.port_id` 必填
- `stores.port_id` 仅允许写入一次
