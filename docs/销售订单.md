# 销售订单

## 1. 文档目标

定义码头版销售订单模型，明确按人聚合与按商品明细的层级关系。

---

## 2. 领域主链路

当前稳定链路：

`port -> store -> ship -> voyage -> person_sale_order -> item_sale_order`

说明：

- 销售记录必须归属航次
- 通过航次和船只归属到店铺与港口
- 普通用户版仅展示自身相关订单视图

---

## 3. 核心模型

### 3.1 按人销售订单 `person_sale_order`

用于承载整单维度：

- 订单归属（port/store/ship/voyage/user）
- 应收、已付、未付
- 付款状态

### 3.2 按商品销售明细 `item_sale_order`

用于承载商品行：

- 商品信息
- 单价、数量、行金额
- 归属字段快照

---

## 4. 当前已确认规则

1. 码头版以按人销售订单为主聚合单元
2. 一个按人订单下可有多个按商品明细
3. 普通用户版只展示与当前用户相关订单
4. 付款状态由服务端基于有效流水汇总得出
5. 销售订单查询和导出必须校验 `store_id` 与 `voyage_id`

---

## 5. 数据落库建议

建议至少拆分集合：

- `person_sale_orders`
- `item_sale_orders`
- `collection_records`

索引建议：

- `person_sale_orders`：`(store_id, voyage_id, created_at desc)`
- `person_sale_orders`：`(store_id, user_id, created_at desc)`
- `item_sale_orders`：`(person_sale_order_id, created_at asc)`
- `collection_records`：`(person_sale_order_id, created_at asc)`

---
