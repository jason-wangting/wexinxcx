# 店铺管理

## 1. 文档目标

定义码头版中店铺的边界、层级归属、成员关系与数据隔离规则。

---

## 2. 店铺定位

- 店铺是码头版核心经营单元
- 店铺必须归属于一个港口
- 店铺下承载船只、航次与销售数据

主链路：

`port -> store -> ship -> voyage -> person_sale_order -> item_sale_order`

---

## 3. 核心规则

1. 店铺创建时必须选择港口
2. 港口归属一旦选中不可修改
3. 一个用户可加入多个店铺
4. 一个店铺可有多个成员
5. 店铺之间数据必须完全隔离
6. 店铺域访问必须带 `current_store_id`

---

## 4. 店铺成员关系

- 成员关系通过 `store_members` 维护
- 同一用户在同一店铺仅一条有效关系
- 角色在店铺内生效，不跨店继承

---

## 5. 数据结构建议

### 5.1 `stores`

```js
{
  _id: string,
  store_id: string,
  port_id: string,
  name: string,
  avatar: string,
  contact_name: string,
  contact_phone: string,
  status: string, // ACTIVE | INACTIVE | SUSPENDED
  created_by: string,
  created_at: Date,
  updated_at: Date
}
```

索引建议：

- `port_id`
- `status`

### 5.2 `store_members`

```js
{
  _id: string,
  store_id: string,
  user_id: string,
  role: string, // store_admin | member
  status: string, // ACTIVE | PENDING | DISABLED | REMOVED
  invited_by: string,
  invite_id: string,
  joined_at: Date,
  created_at: Date,
  updated_at: Date
}
```

索引建议：

- 唯一索引 `(store_id, user_id)`
- 索引 `(user_id, status)`
- 索引 `(store_id, status)`

---

## 6. 鉴权基线

云函数必须按以下顺序校验：

1. 用户身份有效
2. 用户在目标 `store_id` 存在有效成员关系
3. 角色允许当前操作
4. 目标资源归属同一 `store_id`
